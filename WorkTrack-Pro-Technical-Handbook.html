Project Overview

WorkTrack Pro is a professional-grade Windows desktop application for daily employee attendance tracking, designed for small-to-medium teams (up to ~50 employees) operating from a single office. It replaces paper registers and simple spreadsheets with a cloud-backed, real-time system that provides GPS-verified check-in, leave management, attendance reports, and live admin oversight.

What Problem Does It Solve

The core problems being solved are: (1) verifying that employees are actually at the office when they check in, (2) giving management real-time visibility of who is present vs WFH vs absent, (3) eliminating manual attendance registers and Excel sheets, and (4) providing an organised leave request and approval workflow.

Who Uses It

RoleAccess LevelKey Actions
EmployeeOwn data onlyCheck in/out, apply for leave, submit corrections, view own calendar
AdminAll employee dataLive overview, approve leaves/corrections, generate reports, manage settings
💡

The first account registered is automatically promoted to Admin via a PostgreSQL trigger. All subsequent accounts are created as Employees. Additional admins can be promoted via the Employees panel.

Session Management

Sessions are kept in memory only (persistSession: false). There is no token written to disk. Every app launch requires a fresh login. The JWT is stored in the Supabase client instance and automatically refreshed while the app is running.

Security Model

LayerMechanism
AuthenticationSupabase GoTrue (bcrypt password hashing, JWT tokens)
AuthorisationRow Level Security policies at the PostgreSQL layer — cannot be bypassed from the client
IPC SecuritycontextIsolation: true — renderer has no direct Node access. Only methods exposed via contextBridge in preload are available.

GPS & Location Detection

Location detection is the most critical feature of the app. Desktop Windows machines typically have no GPS chip, requiring a multi-stage fallback strategy.

Detection Stages (in order)

StageMethodTypical AccuracyWhen Used
1Browser navigator.geolocation (low accuracy)30–300 mFirst attempt, 8s timeout
2Windows Location API via Python/winsdk (IPC)50–200 kmIf stage 1 fails (code 2 or 3)
3IP Geolocation via ip-api.comCity-level (~5–50 km)If stage 2 fails
4WiFi SSID detection (IPC → netsh)N/A (yes/no)If all geo fails — gives IP coords for map

Office Detection Logic

At check-in, the app combines two signals to determine if the employee is in the office:

// Both signals checked independently
gpsHit  = haversineDistance(userLat, userLon, officeLat, officeLon) <= officeRadiusM
wifiHit = currentSSID is in the office_wifi_ssid list (from settings)

status  = (gpsHit OR wifiHit) ? "in_office" : "wfh"
✅

Office WiFi SSID is the recommended primary method for desktop PCs. Configure it in Admin → Settings → Office. If an employee is connected to the office network, they are marked In Office regardless of GPS accuracy. No GPS hardware required.

is_late Calculation

Uses Intl.DateTimeFormat to get precise IST hour and minute, then compares as plain integers — avoiding all JavaScript timezone parsing bugs:

nowH, nowM = Intl.DateTimeFormat('en-GB', { timeZone: 'Asia/Kolkata', hour12: false }).formatToParts(new Date())
is_late  = (nowH * 60 + nowM) > (startHour * 60 + startMin + gracePeriodMins)

Daily Reminder Logic

Runs from the admin's machine. On every 5-minute tick while AdminLayout is mounted, it checks: (1) reminders enabled, (2) current IST time ≥ configured reminder time, (3) not already sent today (checked via localStorage key wt_reminder_sent_YYYY-MM-DD). If all conditions met, fetches employees without check-in and sends each one an email.

Auto-Update System (How to Implement)

Auto-updates use electron-updater pointing to GitHub Releases as the update server. This is the standard pattern for Electron apps and is completely free.

Implementation Overview

StepWhat to Do
1. Installnpm install electron-updater in worktrack-electron
2. package.jsonAdd "publish": {"provider": "github", "owner": "Kunallele01", "repo": "worktrack-pro"} inside the build block
3. main.cjsImport autoUpdater from electron-updater. Call autoUpdater.checkForUpdatesAndNotify() after window is created.
4. IPCForward update-available and update-downloaded events to renderer. Add installUpdate IPC handler that calls autoUpdater.quitAndInstall().
5. PreloadExpose onUpdateAvailable, onUpdateDownloaded, installUpdate via contextBridge
6. UIAdd a sticky banner component: "New version ready — Restart to update" with an "Update Now" button

Version Blocking (Force Update)

To prevent old versions from operating:

  1. Add min_app_version key to app_settings in Supabase (e.g. 2.1.0)
  2. On app startup (after login), compare app.getVersion() with min_app_version
  3. If current version is lower, show a full-screen blocking modal: "This version is outdated. Please update to continue."
  4. The "Update Now" button triggers autoUpdater.downloadUpdate() then autoUpdater.quitAndInstall()

Publishing a New Release

  1. Bump version in package.json (e.g. 2.0.0 → 2.1.0)
  2. Run npm run package — builds the installer
  3. Go to github.com/Kunallele01/worktrack-pro → Releases → Create new release
  4. Upload release/WorkTrack Pro Setup 2.1.0.exe and release/WorkTrack Pro Setup 2.1.0.exe.blockmap
  5. Add release notes and publish
  6. Update min_app_version in Supabase if you want to force everyone to update
  7. Optionally use the "Send Update Notification" feature in admin Settings to email all employees
💡

electron-updater downloads the update silently in the background. Employees won't see a large download — just a notification when it's ready to install. The install takes ~10 seconds and requires an app restart.

winsdk GPS → Browser geolocationnavigator.geolocation with WiFi fallbackWebView2 (Chromium) has better location source integration than direct winsdk on many machines.
Resend → SMTPnodemailer + SMTPNo new account needed. Works with existing Gmail/Outlook. Familiar to non-technical admins.
Admin-creates accounts → Self-registerSelf-registrationFor a 50-person startup, admin-gating adds friction with no meaningful security benefit.
Session persistence → No persistenceMemory-only sessionsForces daily login. Reinforces attendance discipline. Avoids stale-token edge cases.
ELECTRON_RUN_AS_NODE issuescripts/launch.js workaroundVS Code sets this env var for its own Electron runtime. It must be cleared before spawning the app or Electron runs as plain Node.js (no API, no GUI).
crossorigin attr issuescripts/fix-html.jsVite adds crossorigin for CORS on HTTP servers. file:// doesn't support CORS, causing scripts to silently fail. Post-build removal is the standard Electron fix.
