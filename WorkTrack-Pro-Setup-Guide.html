Building the .exe Installer

This produces a WorkTrack Pro Setup.exe Windows installer. Employees run this once to install the app on their PC. The installer is roughly 150–200 MB and bundles everything — no Python, no Node, nothing else needed.

Step 1 — Open a terminal in the project folder

Open Windows Terminal or PowerShell (not VS Code's terminal — that sets an environment variable that interferes with Electron). Navigate to the project:

cd "C:\Users\KUNAL LELE\Desktop\Attendance App 2\worktrack-electron"

Step 2 — Run the build command

npm run build

# This does two things:
# 1. Compiles the React app (the UI) into optimised JS/CSS
# 2. electron-builder packages everything into a Windows installer
⏱

The first build takes 3–6 minutes. Subsequent builds are faster. A progress bar will appear in the terminal.

Step 3 — Find the installer

After the build finishes, the installer will be at:

worktrack-electron\release\WorkTrack Pro Setup 2.0.0.exe

This is the only file you need to distribute. Everything else (Chromium, Node runtime, all npm packages) is bundled inside it.

✅

What employees receive: A single WorkTrack Pro Setup 2.0.0.exe file (~160 MB). They double-click it, it installs silently in their user profile (no admin rights required), and creates a desktop shortcut.

Rebuild after updates

Whenever you change the code and want to push an update to employees, run npm run build again and distribute the new installer. Employees run the new installer over the old one — it updates cleanly.

Admin First-Time Configuration

These steps only need to be done once, by you, before distributing to employees.

  1. 1
    Install and open the app on your machine.
    Run the installer you just built. Launch "WorkTrack Pro" from the desktop shortcut.
  2. 2
    Register the first account — this becomes the admin.
    Click "Register" and fill in your details. The very first account registered is automatically promoted to Admin. Use your own name and email.
  3. 3
    Sign in and go to Settings → Office Location & Hours.
    Your office coordinates are already pre-filled (18.460204, 73.798938 — Pune). Adjust the radius slider if needed (default: 200 m). Set the office start time (default: 09:30).
  4. 4
    Set your office WiFi network name.
    In "Office WiFi Network Names", type the exact name (SSID) of your office WiFi — e.g. OfficeWiFi. This is the most reliable way to detect who is in the office on a desktop PC. If your WiFi has multiple bands (2.4 GHz and 5 GHz), add both names, one per line.
  5. 5
    Set your company name.
    Change "Your Company" to your actual startup name. It appears in the sidebar and reports.
  6. 6
    Click "Save Office Settings".
    These settings are stored in the cloud and are shared with everyone automatically.
  7. 7
    (Optional) Configure email notifications.
    Go to Settings → Email. Enter your Gmail SMTP credentials to get WFH alerts and reports by email. See Section 6 for Gmail setup instructions.
🔑

Admin account is permanent. You can later promote other employees to Admin from the Employees tab. Having two admins is recommended (you + one other person).

Distributing to Employees

Each employee needs to:

  1. 1
    Receive the installer file.
    Share WorkTrack Pro Setup 2.0.0.exe via WhatsApp, email attachment, Google Drive, or a shared folder on the office network. The file is ~160 MB.
  2. 2
    Run the installer.
    Double-click the .exe. Windows may show a "Windows protected your PC" warning — click "More info" then "Run anyway". Installation takes under 30 seconds and requires no admin rights.
  3. 3
    Launch WorkTrack Pro from the desktop shortcut.
    A "WorkTrack Pro" icon appears on the desktop automatically.
  4. 4
    Click "Register" and create their account.
    Each employee enters their own Employee ID, full name, company email, department, and a password. They choose their own password — you never see it.
  5. 5
    Sign in and start using it.
    Once registered and signed in, the dashboard is fully functional. The Check-In button becomes active once the app detects their location.
⚠️

Windows SmartScreen warning: Because the installer isn't from a paid code-signing certificate, Windows shows a blue "Windows protected your PC" screen. Employees must click "More info" and then "Run anyway". This is safe — it's just Windows being cautious about unsigned software.

Employee Setup (per PC)

Each employee needs to do these two things on their PC after installing the app. Takes about 2 minutes.

Enable Windows Location Services

This lets the app detect whether they're at the office or working from home.

  1. 1
    Press Windows key → type "Location privacy settings" → press Enter
  2. 2
    Turn "Location services" → ON
  3. 3
    Turn "Let apps access your location" → ON
  4. 4
    Scroll down and turn "Let desktop apps access your location" → ON
📍

How location detection works: The app uses three methods in order — browser geolocation, Windows Location API, and finally IP-based location. Even on desktop PCs without GPS hardware, the app always resolves a location. Employees connected to the office WiFi (the network name you configured in Settings) are automatically marked as "In Office" regardless of GPS accuracy.

Enable Auto-Start (Recommended)

So the app launches automatically every morning when Windows starts:

  1. 1
    Open WorkTrack Pro → click Settings in the sidebar
  2. 2
    Under "System", toggle "Launch on Windows startup" → ON
✅

Once auto-start is on, the app launches with Windows, minimises to the system tray, and is ready for check-in every morning without the employee needing to think about it.

Network & Security Requirements

Internet Connectivity

WorkTrack Pro requires an internet connection to function. All data is stored in the cloud (Supabase, hosted in Mumbai/South Asia region).

FeatureRequires Internet?Notes
Login / RegisterYesAuthentication via Supabase
Check-In / Check-OutYesWrites to cloud database in real-time
Admin OverviewYesLive data via WebSocket connection
Reports (Excel download)YesFetches data then generates Excel locally
Location detectionPartiallyWiFi SSID check is local; IP geolocation needs internet

Firewall / Proxy Considerations

If your office uses a firewall or proxy, ensure outbound HTTPS (port 443) is allowed to:

*.supabase.co          # Database and authentication
ip-api.com              # IP geolocation fallback (optional)
unpkg.com               # Map tile CDN (only used in Map tab)

(Optional) Gmail SMTP for Email Notifications

To receive WFH alerts and send monthly reports by email:

  1. 1
    Go to myaccount.google.com → Security → 2-Step Verification → make sure it's ON
  2. 2
    Still in Security, scroll to App passwords → create one named "WorkTrack" → copy the 16-character password
  3. 3
    In WorkTrack Pro admin → Settings → Email: set Host = smtp.gmail.com, Port = 587, Your email, and paste the app password
  4. 4
    Click "Send Test Email" to confirm it works
⚠️

Use a Google App Password, not your regular Gmail password. App passwords are 16 characters with no spaces and work even with 2FA enabled.

Troubleshooting Reference

Problem Likely Cause Solution
"Cannot connect to server" No internet connection Check WiFi / ethernet. The app needs internet to work.
Login shows blank / nothing loads First launch, slow connection Wait 10 seconds. If still blank, close and reopen the app.
"Location Unavailable" — check-in greyed out Windows Location Services off Settings → Privacy → Location → turn everything ON. Then click "Refresh" in the app.
Checked in as WFH when at office Office WiFi SSID not configured, or GPS inaccurate Admin must set the office WiFi name in Settings. Once set, being connected to office WiFi automatically means "In Office".
"Already checked in today" Check-in already recorded Normal — you can only check in once per day. Use Check-Out when leaving.
Windows SmartScreen blocks installer No code signing certificate Click "More info" → "Run anyway". This is safe. It only appears because the installer isn't from a paid certificate.
App doesn't start on Windows startup Startup toggle wasn't set Open the app → Settings → System → toggle "Launch on Windows startup" ON.
Emails not sending SMTP not configured or wrong password Admin: Settings → Email → verify SMTP host, port, and App Password. Click "Send Test Email".
Forgot password — Login screen → "Forgot password?" → enter email → a 6-digit code is sent → enter code → set new password.
Employee account deactivated Admin deactivated the account Admin → Employees tab → find the user → click "Activate".
App shows "GPU Cache Creation failed" Harmless error — VS Code also uses Electron Ignore. Doesn't affect functionality. Disappears on employee machines (they won't have VS Code running).
